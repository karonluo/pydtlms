# 博士生生命周期管理系统需求覆盖率评估报告

## 0. 评估说明

- 评估时间：2026-04-03
- 评估对象：当前仓库主干代码、前端已上线页面、后端已注册 API 与已落地流程能力
- 评估基线：原始需求文档与需求分析报告中定义的一级功能模块、主业务流程与关键审批机制

## 1. 评估范围与口径

本次评估以需求分析文档中“四、功能模块体系设计”提出的 10 个一级功能模块为主口径，结合当前前后端代码实现情况进行比对。

评估规则如下：

- 已覆盖：核心功能已形成前后端可用模块，记 1 分
- 部分覆盖：仅实现部分子功能、仅有基础数据结构、或缺少关键流程闭环，记 0.5 分
- 未覆盖：未见独立功能模块或实际可用能力，记 0 分

模块覆盖率计算公式：

- 覆盖率 = 已实现得分总和 / 10

## 2. 总体结论

按“一级功能模块”口径评估，当前项目对需求的覆盖率仍约为 **70%**。

- 已覆盖：6 个模块
- 部分覆盖：3 个模块
- 未覆盖：1 个模块

折算得分：

- 6 × 1 + 3 × 0.5 + 1 × 0 = 7.0
- 模块覆盖率 = 7.0 / 10 = **70%**

作为补充，从需求文档中的 8 项主要业务流程口径看，当前流程覆盖度约为 **62.5%**。相较此前版本，当前系统的审批流闭环、角色动作约束、业务状态托管与处理历史留痕已显著增强，但尚未补齐若干原始需求中缺失的独立模块，因此覆盖率结论未发生档位变化。

## 5. 关键判断说明

### 5.1 已覆盖模块

以下模块已形成明确前后端闭环，具备业务页面、接口、数据结构与基础管理能力：

- 学生主数据管理
- 招生管理
- 外出研修管理
- 权限与安全管理
- 数据分析与可视化
- 招生、学生、培养、学位、系统、流程审批等主干业务域
- 招生申请、科研报告、外出研修、论文主档已实现角色驱动审批动作、流程历史与状态回写

### 5.2 部分覆盖模块

以下模块已有基础能力，但尚未达到需求文档描述的完整范围：

- 培养过程管理：缺少师生互评、成长档案等扩展子模块
- 导师关系管理：缺少独立的导师关系确认、导师变更审批流、权限联动页面
- 学位与毕业管理：学位阶段已覆盖，且论文主档已纳入流程驱动；但毕业离校未见落地
- 通知与提醒系统：后端已有提醒任务基础，但前端消息中心、提醒策略配置、通知闭环不足

### 5.3 未覆盖模块

以下模块当前代码中未见独立实现：

- 科研成果管理：未见成果登记、论文/专利/竞赛成果归档、统计分析等功能模块

## 6. 业务流程补充评估

需求文档定义了 8 项主要业务流程。按同样的“已覆盖/部分覆盖/未覆盖”口径评估，当前结果如下：

| 流程 | 状态 | 说明 |
| --- | --- | --- |
| 招生录取流程 | 已覆盖 | 已有招生计划、申请池、阶段推进，且关键节点已纳入流程动作驱动 |
| 导师关系确认流程 | 部分覆盖 | 有导师与团队基础数据，缺少独立确认/审批流 |
| 培养方案制定流程 | 已覆盖 | 已有培养方案管理 |
| 科研报告提交与审阅流程 | 已覆盖 | 已有科研报告管理，且已形成审阅动作与历史留痕 |
| 外出研修申请流程 | 已覆盖 | 已有外出研修管理，且已形成审批动作闭环 |
| 学位论文答辩流程 | 部分覆盖 | 已有论文、盲审、状态字段，论文主档已纳入流程驱动；完整答辩组织链路不足 |
| 奖助评定流程 | 未覆盖 | 未见对应模块 |
| 毕业离校流程 | 未覆盖 | 未见对应模块 |

折算后流程覆盖率：

- 4 × 1 + 2 × 0.5 + 2 × 0 = 5.0
- 流程覆盖率 = 5.0 / 8 = **62.5%**

## 7. 结论与建议

当前项目已经完成了需求中的主干业务骨架，特别是：

- 招生
- 学生主档
- 培养管理
- 学位管理
- 审批中心
- 系统治理
- 数据看板

同时，最近版本已进一步补强以下能力：

- 审批任务统一走流程驱动，不再通过普通新增/修改直接改写托管状态
- 审批动作已按当前角色过滤可执行项
- 任务详情支持处理动作、处理意见与处理历史查看
- 运行态已同步至 Flowable 风格兼容表，为后续 BPMN 扩展预留基础

因此从“可运行业务系统”角度看，主体框架已经具备，需求实现程度不低；从“严格对照原始需求逐项验收”角度看，当前仍属于“主干模块基本完成，扩展模块与补充闭环待完善”的阶段。

但如果严格按原始需求文档口径验收，仍有以下短板会拉低覆盖率：

- 科研成果管理缺失
- 导师关系管理未形成独立闭环
- 通知提醒系统缺少前台能力与完整配置面
- 毕业离校流程未落地
- 奖助评定流程未落地

综合判断：

- **按功能模块口径：当前覆盖率约 70%**
- **按业务流程口径：当前覆盖率约 62.5%**

若以项目阶段来定义，可判断当前处于：

- **“主干模块已建成，扩展模块和闭环流程待补齐”** 的状态。
