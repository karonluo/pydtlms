# 博士生生命周期管理系统单元测试用例

## 1. 说明

- 测试目标：验证本轮已完成缺陷修复与关键增强功能在代码层面的稳定性，避免后续回归导致后端启动阻塞、超级管理员权限失效或 PostgreSQL 运行态初始化重复执行。
- 测试范围：仅覆盖本轮已完成且适合单元测试的后端逻辑，不涉及真实 PostgreSQL、Redis、浏览器或端到端页面交互。
- 测试方式：基于 pytest，使用 mock 和 monkeypatch 隔离外部依赖。

## 2. 测试环境

- 操作系统：Windows
- Python：3.12
- 测试框架：pytest
- 被测模块：backend/app/core/rbac.py、backend/app/services/management_service.py、backend/app/services/postgres_state_store.py

## 3. 测试用例

| 用例编号 | 用例名称 | 前置条件 | 测试步骤 | 预期结果 |
| --- | --- | --- | --- | --- |
| UT-001 | 超级管理员通配权限放行 | 构造 permissions 包含 * 的 Principal | 调用 require_permissions('dashboard:view') 返回的依赖函数 | 不抛出异常，直接返回当前 Principal |
| UT-002 | 普通用户缺少权限时拒绝访问 | 构造不含目标权限的 Principal | 调用 require_permissions('dashboard:view') 返回的依赖函数 | 抛出 403 异常，错误信息包含缺失权限 |
| UT-003 | 懒加载代理在属性访问前不实例化真实 Store | 替换 RuntimeManagementStore 为可计数假对象 | 创建 LazyRuntimeManagementStore 实例但不访问任何属性 | 真实 RuntimeManagementStore 构造次数为 0 |
| UT-004 | 懒加载代理仅初始化一次并复用实例 | 替换 RuntimeManagementStore 为可计数假对象 | 连续访问 LazyRuntimeManagementStore 的两个属性 | 真实 RuntimeManagementStore 仅构造 1 次，属性调用均来自同一实例 |
| UT-005 | PostgreSQL 连接串包含 connect_timeout | 创建 PostgresStateStore 实例 | 调用 _build_dsn('db_dtlms', 'postgres') | 返回的 DSN 包含 host、port、dbname、user、password 和 connect_timeout=5 |
| UT-006 | schema 探测在运行态表存在时返回已初始化 | 构造返回非空 regclass 的假游标 | 调用 _schema_initialized(cur) | 返回 True |
| UT-007 | 已初始化 schema 时跳过 SQL 初始化脚本执行 | mock _connect、ensure_database、_schema_initialized | 调用 ensure_schema() | 不执行任何 SQL 文件读取或 SQL 初始化语句，_schema_ready 置为 True |
| UT-008 | 未初始化 schema 时按顺序执行初始化 SQL 脚本 | mock _connect、ensure_database、_schema_initialized 与 SQL 文件内容 | 调用 ensure_schema() | 依次执行 SQL_FILES[1:] 中各脚本内容，并将 _schema_ready 置为 True |

## 4. 通过准则

- 所有 UT 编号用例执行结果均为通过。
- pytest 退出码为 0。
- 不依赖真实数据库和缓存即可重复执行。

- 本次单元测试聚焦于已完成修复的根因逻辑，后续如需补充接口级回归，可在此基础上增加 API 测试与集成测试。
