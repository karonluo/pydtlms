# 生产环境学生模块列式化改造上线步骤

## 1. 目的

本文件用于指导学生模块正式表收口上线，覆盖以下对象：

1. 学生主档 `dtlms_students`
2. 研究中心/团队 `dtlms_teams`
3. 团队导师关系 `dtlms_team_advisors`
4. 学生个人空间 `dtlms_user_profiles`
5. 门户注册学生 `dtlms_portal_students`

## 2. 上线前提

1. 已完成生产库全量备份。
2. 已确认后端代码包含 [backend/sql/029_student_team_runtime_backfill.sql](backend/sql/029_student_team_runtime_backfill.sql) 与 [backend/scripts/backfill_students_and_teams_to_relational.py](backend/scripts/backfill_students_and_teams_to_relational.py)。
3. 已确认本次上线不涉及 Flowable 托管对象状态迁移。

## 3. 推荐执行顺序

1. 先部署后端代码。
2. 先执行 dry-run。
3. 再执行正式补数。
4. 最后执行核查 SQL。

## 4. dry-run 命令

```powershell
pwsh -File backend/scripts/update_student_team_fix_production.ps1
```

如需只检查部分数据，可附加：

```powershell
pwsh -File backend/scripts/update_student_team_fix_production.ps1 -TeamLimit 20 -StudentLimit 50
```

## 5. 正式执行命令

```powershell
pwsh -File backend/scripts/update_student_team_fix_production.ps1 -Apply
```

## 6. 必做核查 SQL

```sql
SELECT COUNT(*) AS runtime_team_count FROM dtlms_runtime_teams;
SELECT COUNT(*) AS relational_team_count FROM dtlms_teams WHERE is_deleted = FALSE;

SELECT COUNT(*) AS runtime_student_count FROM dtlms_runtime_students;
SELECT COUNT(*) AS relational_student_count FROM dtlms_students WHERE is_deleted = FALSE;

SELECT id, team_code, team_name, department_name, team_status, updated_at
FROM dtlms_teams
WHERE is_deleted = FALSE
ORDER BY updated_at DESC, id DESC
LIMIT 20;

SELECT id, student_no, full_name, current_status, enrollment_year, updated_at
FROM dtlms_students
WHERE is_deleted = FALSE
ORDER BY updated_at DESC, id DESC
LIMIT 20;
```

## 7. 风险控制

1. 本次脚本默认只读，只有显式加 `-Apply` 才会写入。
2. 补数脚本为幂等实现，可重复执行。
3. 本次不删除 runtime JSONB 表，仅把其降级为兼容镜像。
4. 如发现异常，应先停止继续写操作，保留现场，再按备份回滚。
