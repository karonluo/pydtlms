# 招生系统-学生填报内容

## 1. 文档目的

本文基于 [需求/原始需求资料/招生系统字段.xlsx](需求/原始需求资料/招生系统字段.xlsx) 对学生填报页进行结构化分析，输出后续替换 `/portal/application` 所需的实体拆分、属性定义、实体关系与关键业务规则。

本文定位不是界面原型，而是开发基线。后续前端、后端、数据库和导入导出结构均应以本文件为准统一建模，替换当前门户中“单表 + 大文本字段”的实现方式。

## 2. 数据来源与现状结论

### 2.1 字段来源

Excel 原表实际采用四列结构：

| 列         | 含义                                   |
| ---------- | -------------------------------------- |
| 档案主目录 | 页面一级章节                           |
| 二级目录   | 字段名或子项名                         |
| 填写描述   | 候选值、示例值、说明文本               |
| 备注       | 控件类型、是否必填、联动规则、上传规则 |

### 2.2 原表可识别的章节

| 主目录                       | 说明                             |
| ---------------------------- | -------------------------------- |
| 基本信息                     | 考生身份与联系方式               |
| 报名信息                     | 志愿研究中心、导师志愿、获知渠道 |
| 教育经历                     | 可增删改的多条经历               |
| 实践经历（非必填）           | 可增删改的多条经历               |
| 英语语言能力                 | 可增删改的多条能力记录           |
| 家庭情况                     | 可增删改的多条家庭成员记录       |
| 论文发表及获奖经历（非必填） | 可增删改的多条成果记录           |
| 个人陈述                     | 长文本与简历附件                 |
| 提交申请                     | 进度展示与声明确认               |

### 2.3 当前系统现状

当前 `/portal/application` 及对应接口仍是“主档平铺 + 多段文本字段”模式，尚未按 Excel 拆分为子实体：

| 现状位置                                                                                                              | 当前问题                                                                        |
| --------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------- |
| [frontend/src/views/portal/StudentPortalApplicationView.vue](frontend/src/views/portal/StudentPortalApplicationView.vue) | 页面只支持少量主档字段，缺少多实体子表单                                        |
| [frontend/src/api/portal.ts](frontend/src/api/portal.ts)                                                                 | `education_experience`、`practice_experience`、`family_info` 等仍为字符串 |
| [backend/app/schemas/portal.py](backend/app/schemas/portal.py)                                                           | `PortalApplicationUpsert` 仍是单对象扁平结构                                  |
| [backend/app/api/v1/portal.py](backend/app/api/v1/portal.py)                                                             | 门户提交接口尚未承载结构化子实体                                                |
| [backend/app/services/management_service.py](backend/app/services/management_service.py)                                 | 门户申请保存逻辑仍按平铺字段写入                                                |

结论：后续替换 `/portal/application` 时，不能只改前端页面，必须同步重构门户 DTO、后端存储模型与 PostgreSQL 物理表结构。

## 4. 实体详细设计

### 4.1 PortalApplicantProfile 考生填报主档

说明：该实体承接“基本信息”中的 1:1 字段。它与门户注册账号强关联，其中姓名、身份证号、手机号、邮箱优先来自注册信息；邮箱在 Excel 中明确要求从注册信息带入报名填写界面。

| 属性名                      | 中文名           | 类型建议     | 必填 | 说明                       |
| --------------------------- | ---------------- | ------------ | ---- | -------------------------- |
| id                          | 主键             | bigint       | 是   | 系统主键                   |
| portal_student_id           | 门户学生ID       | bigint       | 是   | 对应门户账号               |
| full_name                   | 姓名             | varchar(64)  | 是   | 注册信息同步               |
| full_name_pinyin            | 姓名拼音         | varchar(128) | 否   | 文本输入                   |
| gender                      | 性别             | varchar(16)  | 是   | 枚举：男/女                |
| native_place                | 籍贯             | varchar(128) | 否   | 文本输入                   |
| ethnic_group                | 民族             | varchar(32)  | 是   | 中国民族字典               |
| political_status            | 政治面貌         | varchar(32)  | 是   | 字典单选                   |
| id_type                     | 证件类型         | varchar(32)  | 是   | 中国常用证件类型字典       |
| id_number                   | 证件号码         | varchar(64)  | 是   | 注册信息同步               |
| id_card_attachment_id       | 证件上传附件ID   | bigint       | 否   | 证件照正反面               |
| email                       | 邮箱             | varchar(128) | 是   | 从注册信息带入             |
| phone_number                | 手机号           | varchar(32)  | 是   | 注册信息同步               |
| mailing_address             | 通讯地址         | varchar(255) | 否   | 文本输入                   |
| emergency_contact_name      | 紧急联系人姓名   | varchar(64)  | 否   | 文本输入                   |
| emergency_contact_phone     | 紧急联系人手机号 | varchar(32)  | 否   | 文本输入                   |
| profile_photo_attachment_id | 个人照片附件ID   | bigint       | 否   | JPEG/JPG/PNG，1MB 内证件照 |

### 4.2 PortalApplication 招生报名申请

说明：该实体承接一次正式报名，是所有子实体的根对象。每个考生可在不同招生计划下有不同申请，但同一计划下应限制为一条有效申请主档。

| 属性名               | 中文名                | 类型建议     | 必填 | 说明                              |
| -------------------- | --------------------- | ------------ | ---- | --------------------------------- |
| id                   | 主键                  | bigint       | 是   | 系统主键                          |
| applicant_profile_id | 考生填报主档ID        | bigint       | 是   | 关联 `PortalApplicantProfile`   |
| plan_id              | 招生计划ID            | bigint       | 是   | 关联 `RecruitmentPlan`          |
| business_key         | 业务编号              | varchar(64)  | 是   | 与招生申请流程主键保持一致        |
| application_status   | 申请状态              | varchar(32)  | 是   | 草稿、已提交、审核中等            |
| source_channel       | 了解项目方式          | varchar(64)  | 是   | 单选枚举                          |
| source_channel_other | 了解项目方式-其他说明 | varchar(255) | 否   | 当 `source_channel=其他` 时必填 |
| current_step_code    | 当前步骤编码          | varchar(32)  | 否   | 前端流程展示可派生，也可冗余      |
| completion_percent   | 总完成度              | numeric(5,2) | 否   | 可派生，建议不落库或作为缓存字段  |
| submitted_at         | 提交时间              | timestamp    | 否   | 正式提交时生成                    |
| created_at           | 创建时间              | timestamp    | 是   | 系统时间                          |
| updated_at           | 更新时间              | timestamp    | 是   | 系统时间                          |

### 4.3 PortalApplicationPreference 报名志愿

说明：Excel 实际是“第一志愿/第二志愿”两组字段。建模时不建议固定成两列，而应拆成带 `preference_order` 的子表，便于后续扩展第三志愿或调剂逻辑。

| 属性名               | 中文名           | 类型建议     | 必填 | 说明                         |
| -------------------- | ---------------- | ------------ | ---- | ---------------------------- |
| id                   | 主键             | bigint       | 是   | 系统主键                     |
| application_id       | 报名申请ID       | bigint       | 是   | 关联 `PortalApplication`   |
| preference_order     | 志愿顺序         | int          | 是   | 1=第一志愿，2=第二志愿       |
| research_center_id   | 意向研究中心ID   | bigint       | 是   | 顺序 1 必填，顺序 2 选填     |
| research_center_name | 意向研究中心名称 | varchar(128) | 是   | 冗余展示字段                 |
| advisor_id           | 意向导师ID       | bigint       | 否   | 从所选研究中心下导师列表选择 |
| advisor_name         | 意向导师姓名     | varchar(64)  | 否   | 冗余展示字段                 |
| is_optional          | 是否选填志愿     | boolean      | 是   | 第二志愿为 true              |

关键规则：

| 规则         | 说明                                       |
| ------------ | ------------------------------------------ |
| 第一志愿必填 | 第一志愿必须选择研究中心                   |
| 第二志愿选填 | 第二志愿可为空                             |
| 研究中心单选 | 每个志愿只能选择一个研究中心               |
| 导师联动     | 导师候选集来自该研究中心下的导师团队       |
| 默认导师     | 可直接选团队负责人，默认负责人作为意向导师 |

### 4.4 PortalEducationExperience 教育经历

说明：Excel 明确标注为“一个学生对应多个学历，可增删改”，属于标准 1:N 子实体。

| 属性名                           | 中文名              | 类型建议     | 必填 | 说明                                         |
| -------------------------------- | ------------------- | ------------ | ---- | -------------------------------------------- |
| id                               | 主键                | bigint       | 是   | 系统主键                                     |
| application_id                   | 报名申请ID          | bigint       | 是   | 关联 `PortalApplication`                   |
| education_stage                  | 教育阶段            | varchar(32)  | 是   | 硕士毕业、硕士在读且本科毕业、本科在读、高中 |
| start_date                       | 开始年月            | date         | 是   | 起止年月起                                   |
| end_date                         | 结束年月            | date         | 是   | 起止年月止                                   |
| school_id                        | 学校ID              | bigint       | 否   | 可来自学校库                                 |
| school_name                      | 就读学校            | varchar(128) | 是   | 学校库可选或手填                             |
| major_name                       | 就读专业            | varchar(128) | 是   | 文本输入                                     |
| average_score                    | 期间平均成绩        | varchar(64)  | 否   | 原表未限定数值格式，先保留字符型             |
| gpa                              | 期间绩点            | varchar(32)  | 否   | 原表未限定精度                               |
| ranking                          | 平均成绩排名        | varchar(64)  | 否   | 如“5/120”                                  |
| transcript_attachment_id         | 成绩单附件ID        | bigint       | 否   | 高中阶段无需上传，其余阶段必传               |
| degree_certificate_attachment_id | 毕业证/学位证附件ID | bigint       | 否   | 本科及以上毕业阶段必传                       |
| verifier_name                    | 证明人姓名          | varchar(64)  | 否   | 文本输入                                     |
| verifier_phone                   | 证明人手机          | varchar(32)  | 否   | 文本输入                                     |
| sort_order                       | 排序                | int          | 是   | 按最高学历到高中依次填写                     |

关键规则：

| 规则           | 说明                                    |
| -------------- | --------------------------------------- |
| 多条记录       | 一个学生对应多条教育经历                |
| 填写顺序       | 从最高学历到高中依次填写                |
| 高中规则       | 高中经历不上传成绩单                    |
| 本科及以上规则 | 本科及以上需要上传毕业证/学位证和成绩单 |
| 学校来源       | 学校可来自学校主数据，也允许手输        |

### 4.6 PortalEnglishProficiency 英语语言能力

| 属性名                    | 中文名         | 类型建议    | 必填 | 说明                       |
| ------------------------- | -------------- | ----------- | ---- | -------------------------- |

| application_id            | 报名申请ID     | bigint      | 是   | 关联 `PortalApplication` |

### 4.7 PortalFamilyMember 家庭成员

说明：Excel 明确要求“父母必填，兄弟姐妹选填”，且关系类型存在枚举与数量约束。

| 属性名         | 中文名     | 类型建议     | 必填 | 说明                       |
| -------------- | ---------- | ------------ | ---- | -------------------------- |
| id             | 主键       | bigint       | 是   | 系统主键                   |
| application_id | 报名申请ID | bigint       | 是   | 关联 `PortalApplication` |
| member_name    | 姓名       | varchar(64)  | 是   | 家庭成员姓名               |
| relation_type  | 与本人关系 | varchar(16)  | 是   | 父亲、母亲、兄、弟、姐、妹 |
| employer_name  | 工作单位   | varchar(255) | 否   | 文本输入                   |
| job_title      | 职务       | varchar(128) | 否   | 文本输入                   |
| contact_phone  | 联系电话   | varchar(32)  | 否   | 文本输入                   |

关键规则：

| 规则         | 说明                               |
| ------------ | ---------------------------------- |
| 至少两条必填 | 父亲、母亲信息必须具备             |
| 兄弟姐妹选填 | 兄、弟、姐、妹可按实际情况填写多条 |

### 4.8 PortalAchievementRecord 论文/获奖经历

说明：Excel 标注该部分为“非必填”，且把论文发表和获奖经历放在同一组中，建议统一实体，通过 `achievement_type` 做分流。

| 属性名                | 中文名        | 类型建议     | 必填 | 说明                       |
| --------------------- | ------------- | ------------ | ---- | -------------------------- |
| id                    | 主键          | bigint       | 是   | 系统主键                   |
| application_id        | 报名申请ID    | bigint       | 是   | 关联 `PortalApplication` |
| achievement_type      | 类型          | varchar(32)  | 是   | 科研类、学生活动、论文发表 |
| paper_title           | 论文名称      | varchar(255) | 否   | 论文发表时必填             |
| author_order          | 作者序位      | varchar(32)  | 否   | 论文发表时必填             |
| journal_or_conference | 期刊/会议名称 | varchar(255) | 否   | 论文发表时必填             |
| publish_or_index_date | 发表/收录日期 | date         | 否   | 论文发表时必填，按年月     |
| award_name            | 奖项名称      | varchar(255) | 否   | 获奖时填写                 |
| awarding_organization | 颁发机构      | varchar(255) | 否   | 获奖时填写                 |
| award_level           | 获奖等级      | varchar(128) | 否   | 如全国一等奖               |

| 多条记录     | 一个学生可对应多条成果记录                                   |
| 类型字典     | 类型来源于字典：科研类、学生活动、论文发表                   |
| 论文分支必填 | 论文发表时，期刊/会议、作者序位、论文名称、发表/收录日期必填 |

### 4.9 PortalPersonalStatement 个人陈述

说明：个人陈述不是简单一段文本，而是“主陈述 + 两道补充问答 + 简历附件”的组合实体。

| 属性名                  | 中文名          | 类型建议 | 必填 | 说明                    |
| ----------------------- | --------------- | -------- | ---- | ----------------------- |
| application_id          | 报名申请ID      | bigint   | 是   | 与申请一对一            |
| personal_statement_text | 个人陈述正文    | text     | 是   | 800-1200 字，中英文均可 |
| ai_problem_statement    | AI 关键问题思考 | text     | 否   | 选填                    |
| ai_industry_opinion     | AI 行业不同观点 | text     | 否   | 选填                    |
| resume_attachment_id    | 个人简历附件ID  | bigint   | 否   | 支持 Word/PDF           |

### 4.10 PortalApplicationDeclaration 提交确认

说明：该部分包含前端进度显示和最终声明确认。进度属于派生信息，声明勾选属于必须保存的业务状态。

| 属性名               | 中文名           | 类型建议 | 必填 | 说明                         |
| -------------------- | ---------------- | -------- | ---- | ---------------------------- |
| application_id       | 报名申请ID       | bigint   | 是   | 与申请一对一                 |
| has_read_declaration | 已阅读声明并勾选 | boolean  | 是   | 提交前必须为 true            |
| declaration_text     | 声明内容         | text     | 是   | 可静态配置，也可落库留痕     |
| progress_snapshot    | 完成度快照       | jsonb    | 否   | 如需保留提交时的分模块完成度 |

### 4.11 PortalAttachment 通用附件

说明：为避免每个子实体重复建文件表，建议使用统一附件实体，并通过业务归属字段挂接到不同对象。

## 8. 开发建议

### 8.1 建模原则

| 原则             | 说明                                                     |
| ---------------- | -------------------------------------------------------- |
| 先结构化再页面化 | 先定数据库和 DTO，再做页面，不要先拼页面再倒推结构       |

## 9. 最终结论

基于 Excel 字段表，学生填报内容不应继续被实现为单一报名主档，而应至少拆分为 10 个核心实体：考生填报主档、报名申请、报名志愿、教育经历、实践经历、英语语言能力、家庭成员、论文/获奖经历、个人陈述、提交确认，并辅以统一附件实体。

后续替换 `/portal/application` 时，应按本文结构重构前端页面、接口 DTO、后端服务和 PostgreSQL 物理结构；否则将继续出现字段无法扩展、规则难以校验、附件无法归档、导出结构混乱的问题。
